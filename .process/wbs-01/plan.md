# WBS-01: 두렙(DooRep) 데모 사이트 구현 계획

## 프로젝트 개요

**목적:** 파충류 전문 커뮤니티 통합 커머스 플랫폼 "두렙(DooRep)" 제안서의 핵심 기능을 시연하는 데모 사이트 구축

**기술 스택:**
- Framework: Next.js 15 (App Router)
- Styling: Tailwind CSS v4
- Deployment: Cloudflare Workers (OpenNext adapter)
- Language: TypeScript

**총 페이지:** 5개
1. 데모 홈
2. 스타일 가이드
3. 게이미피케이션 - 룰렛
4. 게이미피케이션 - 출석체크
5. 상품 태깅

---

## 단계별 구현 계획

### 1단계: 프로젝트 초기 설정 & 인프라

#### 1-1. Next.js 프로젝트 생성 및 Cloudflare Workers 설정
- Next.js 15 프로젝트 생성 (App Router, TypeScript)
- Tailwind CSS v4 설정
- `@opennextjs/cloudflare` 설치 및 설정
- `wrangler.jsonc` 구성
- `open-next.config.ts` 작성
- 로컬 빌드 검증

#### 1-2. 공통 레이아웃 & 디자인 시스템 구성
- 글로벌 CSS 변수 (컬러 팔레트) 정의
- Tailwind 커스텀 테마 확장
- 공통 레이아웃 컴포넌트 (Header, Footer, Navigation)
- 반응형 사이드바/모바일 네비게이션
- 기본 UI 컴포넌트 (Button, Card, Badge)
- Pretendard / Inter 폰트 설정

### 2단계: 핵심 페이지 구현

#### 2-1. 데모 홈 페이지
- 히어로 섹션 (프로젝트 소개, 브랜드 비주얼)
- 핵심 기능 카드 (상품태깅, 게이미피케이션, 커뮤니티)
- 각 데모 페이지로의 네비게이션 카드
- 제안서 핵심 가치 시각화
- 반응형 레이아웃

#### 2-2. 스타일 가이드 페이지
- 컬러 팔레트 시각화 (색상 스와치 + HEX 코드)
- 타이포그래피 스케일 데모
- 버튼/배지/카드 등 UI 컴포넌트 쇼케이스
- 간격/그리드 시스템 가이드
- 아이콘 세트 미리보기

### 3단계: 게이미피케이션

#### 3-1. 룰렛 페이지
- 시각적 룰렛 휠 UI (CSS/Canvas 기반 회전 애니메이션)
- 파충류 테마 보상 항목 (포인트, 쿠폰, 사료 샘플 등)
- 당첨 결과 모달 + 축하 이펙트
- 일일 1회 참여 제한 UI (데모용 리셋 버튼 포함)
- 당첨 확률 표시 & 보상 목록

#### 3-2. 출석체크 페이지
- 월간 캘린더 뷰 (출석 현황 시각화)
- 출석 도장 애니메이션 (파충류 발자국 테마)
- 연속 출석 보상 표시 (7일/14일/30일 개근 보상)
- 오늘의 출석체크 버튼 + 완료 상태
- 누적 포인트/보상 현황 카드

### 4단계: 상품 태깅

#### 4-1. 상품 태깅 페이지 (오늘의집 UX 차용)
- 사육장 사진 위 좌표 기반 상품 태그(+) 마커 배치
- 마커 클릭 시 상품 정보 팝업 (상품명, 가격, 썸네일)
- 팝업에서 상품 상세 링크 제공
- 3개의 데모 상품 데이터 (사육장, 은신처, 조명 등)
- 사진 3장에 각각 상품 태깅 적용
- 피드 형식의 레이아웃 (오늘의집 '집들이' 스타일)
- 좋아요/댓글 수 표시 (UI only)
- 반응형 대응 (모바일에서 태그 위치 스케일링)

### 5단계: 통합 & 배포

#### 5-1. 최종 통합 및 Cloudflare Workers 배포
- 전체 페이지 네비게이션 연결 확인
- 반응형 테스트 (모바일/태블릿/데스크탑)
- OpenNext 빌드 검증
- Cloudflare Workers 배포
- 배포 후 동작 확인

## 작업 흐름 규칙

각 단계(1-1, 1-2, ... 5-1)마다:

1. **계획 수립:** `wbs-01/{단계}-plan.md` 작성
2. **셀프 리뷰:** 계획의 완성도/실현가능성 검토, 필요시 수정
3. **실행:** 코드 구현
4. **결과 기록:** `wbs-01/{단계}-result.md` 작성
5. **검토:** plan.md 대비 결과 검토, 이슈 있으면 재실행
6. **진행 업데이트:** `wbs-01.md` 체크리스트 갱신
7. **핸드오프:** `handoff.md` 갱신 (세션 복원용)

## 기술적 고려사항

1. **정적 데이터:** 데모이므로 DB 없이 로컬 JSON/상수로 데이터 관리
2. **상태 관리:** localStorage를 활용한 클라이언트 상태 (출석, 룰렛 참여 기록)
3. **이미지:** Unsplash/자체 생성 이미지 또는 placeholder 활용
4. **애니메이션:** CSS transitions + Framer Motion (필요시)
5. **번들 사이즈:** Cloudflare Workers 10MB 제한 고려, 최소 의존성
