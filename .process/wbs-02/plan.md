# WBS-02: 제안서 슬라이드 뷰어 구현 계획

## 프로젝트 개요

**목적:** `docs/proposal.md` 제안서를 브라우저에서 세로 슬라이드 형태로 열람하고, PDF로 다운로드할 수 있는 페이지 구현

**핵심 요구사항:**
1. `docs/proposal.md` 내용을 슬라이드 콘텐츠로 변환 (`docs/slide`)
2. 세로 방향 한 페이지씩 슬라이딩 (scroll-snap)
3. PDF 형식에 맞는 레이아웃 (A4 비율)
4. 브라우저에서 PDF 다운로드 가능

**기술 접근:**
- 슬라이드 데이터: `src/data/slides.ts` (구조화된 TypeScript 데이터)
- 뷰어: `/slides` 페이지 (CSS scroll-snap + A4 비율 카드)
- PDF 다운로드: `@media print` CSS → `window.print()` (추가 라이브러리 불필요)

## 단계별 구현 계획

### 1-1. 슬라이드 데이터 구조화
- `docs/proposal.md` → `src/data/slides.ts` 변환
- 슬라이드별 타입 정의 (title, subtitle, content, layout type)
- 레이아웃 타입: cover, bullets, table, timeline, split

### 1-2. 슬라이드 뷰어 페이지 구현
- `/slides` 라우트 생성
- A4 비율 (210:297) 세로 카드 레이아웃
- CSS `scroll-snap-type: y mandatory` 적용
- 슬라이드 번호 인디케이터
- 키보드 내비게이션 (↑↓, Page Up/Down)
- 기존 디자인 시스템 컬러 활용

### 1-3. PDF 다운로드 기능
- `@media print` CSS로 페이지 브레이크 처리
- 다운로드 버튼 UI
- print 시 네비게이션/인디케이터 숨김
- A4 사이즈 정확한 매핑

### 2-1. 최종 통합 및 빌드 검증
- 헤더 네비게이션에 슬라이드 링크 추가
- 데모 홈에서 슬라이드 접근 경로 추가
- 빌드 검증
- 반응형 확인
